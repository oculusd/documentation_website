What is this all about

OculusD.com, Inc. is a company that provide a platform, services and applications aimed at the IoT and IIoT markets. In essence, this means that any "thing" connected to the Internet could use OculusD to log sensor data. At a later stage, the data can be queried, analysed and processed in ways that will help you understand or monitor your environment and perhaps even learn something new!

The overall solution consists of three parts:

  • Platform - A cloud based platform where all data is aggregated
  • Services - Additional value add services that will be provided by OculusD to query and analyse your data.
  • Applications - The software applications (and libraries) you will need to integrate your IoT/IIoT things with OculusD

Where does what fit in

The cloud platform is where all the RESTFul API end-points are you need to integrate to. It is depicted in the diagram as the "OculusD Service".

Under most circumstances, you will manage your account(s) and things from a Workstation. This is a computer you typically have full access to and can trust as this is where your credentials and profiles will be saved within your environment (don't worry - if you loose anything, it can be re-created from data in the cloud!).

On the workstation, you will typically install several tools:

  • odc_pycommons - The common Python libraries used by all Python based applications integrating to OculusD
  • odc_pyadminlibs - The Python libraries required to perform administrative tasks

From the Workstation, you will also export specific configurations to be installed and/or used on your "things".

On your "thing" (a Raspberry Pi in this example, with a TMP006 Temperature Sensor), additional libraries and applications will be required to collect and send data to the OculusD Service.

On the thing, you will typically install the following tools (based on the example in the image):

  • odc_pycommons - The common Python libraries used by all Python based applications integrating to OculusD
  • odapp_pysysmon - Python based system monitoring (CPU, Memory, Storage and various on board sensors
  • odapp_pysensors - Python based application that uses the configuration generated from the Workstation, with potentially additional libraries, to collect data from sensors and send the data to OculusD

First steps

The basic requirements to get started is to create a root account. A root account is the account that will be billed and can also be used as an administrative account to manage other users and features.

Install the required applications and libraries

The following are pre-requisites for the Workstation which you will use for the administrative functions:

  • A modern operating system, for example Windows 10, Mac OSX or one of the many excellent Linux distributions
  • Internet connectivity, allowing TCP port 443 out to the Internet
  • Python version 3.6+
  • You should be familiar with the command line

Ensure you have a supported Python version:

Try the following command in a terminal (command prompt for Windows users): 

                        
$ python3 --version
Python 3.7.0

Creating a root account

The following example demonstrate the creation of a root account using the Python Libraries. 

                  
    >>> from odc_pyadminlibs.actions.register import root_account_registration
    >>> reg_result = root_account_registration(email_address='your-name@example.tld', passphrase='...a strong passphrase...', account_name='Demo Account', persist_with_passphrase=True)
    >>> reg_result
    {
      'IsError': False, 
      'TraceId': None, 
      'RemoteTraceId': '20181107-7224-8557-2117-5640', 
      'ErrorMessage': None, 
      'Message': 'Root account created...', 
      'RootAccountObj': <odc_pycommons.models.RootAccount object at 0x10a1d9f98>
    }

Required Input Parameters

Parameter Type Description
email_address String A valid e-mail address
passphrase String A passphrase with the following minimum requirements:
  • Minimum Length: 20 characters
  • Maximum Length: 255 characters
Note: It is a good idea to research ways on how to create strong passwords/passphrases. Have a look at this lifehacker.com article if you are new to this concept.
account_name String Free form text field with a human friendly account name

Optional Input Parameters

Parameter Type Description
persist_with_passphrase Boolean A flag to indicate if the passphrase must be persisted as well (default=False)

Output - Function return result

The result is in the form of a Python dictionary, with the following fields:
Parameter Description
IsError This is a boolean type which will only be True if the remote call returned an error or if some other exception occurred
TraceId Echo ot the optional trace id you can supply as input to the function.
RemoteTraceId A trace id returned from the remote service. When you require support it will help a great deal if you can supply the remote trace id.
ErrorMessage If an error was generated, this will typically hold the message generated by that error.
Message A general message from the remote service
RootAccountObj The root account Python object

Output - Configuration Files

When the command executed successfully, it will also create a local configuration directory and file. The default location is in $HOME/.oculusdcli

The file generated will be called root_account and is a SQLite file you can open with any SQLite compatible application/library.

IMPORTANT: Since this file contain credentials, it is strongly recommended that you restrict access to this directory using your operating system's features for these purposes. Here are some quick Google searches to get you going:

  • Windows 10
  • Apple Mac OSX
  • Linux

Output - E-Mail

You should also receive an e-mail with an activation code. This code is required in the next step (activation).

IMPORTANT: The activation code is only valid for 24 hours. If you fail to activate your account in 24 hours you may need to log a support ticket in order to use the same e-mail address again.

Activate a root account

The following example demonstrate the activation of a root account using the Python Libraries. 

                    
      >>> from odc_pyadminlibs.actions.activation import activate_root_account
      >>> act_result = activate_root_account(root_account=reg_result['RootAccountObj'], activation_token='...the activation code you received via email...')
      >>> act_result
      {
        'IsError': False, 
        'TraceId': None, 
        'RemoteTraceId': '20181108-8224-9621-2944-3980', 
        'ErrorMessage': None, 
        'Message': 'Activated root account "...."'
      }

Required Input Parameters

Parameter Type Description
root_account RootAccount A root account that contains the information required for account activation
activation_token String The activation token you received via e-amil

Output - E-Mail

You should also receive an e-mail a final confirmation that your account is now active

Current cloud service status and road map

In a nutshell, this project is in early start-up phase and all code and interfaces should be considered ALPHA version.

Below is a list of features, mapped to the diagram above, with the implementation status as of 2 November 2018. The unfinished items appear in no particular order.

Feature Implemented Description
Root Account Creation Yes Allow new root accounts to be created
Roor Account Activation Yes Allow users to activate their new root account
Root Account Authentication Yes Authenticate against root account credentials and obtain a temporary (24 hour) period token to perform administrative functions
Thing Definition Partial Ability to define a Thing. Currently only an authenticated Root Account can define things
Create Thing Token Partial Use (only) a Root Account to create a thing token. A thing token is required by "things" to send data to the OculusD Cloud
Thing Log Data Yes Log sensor data from a Thing
Delete a Thing Partial (Only) An authenticated Root Account can currently delete a previously defined Thing
Modify A Thing - Add Sensors No Feature has not yet been implemented
Modify A Thing - Delete Sensors No Feature has not yet been implemented
Modify A Thing - Modify Sensors No Feature has not yet been implemented
Query thing and sensor data No Feature has not yet been implemented

Link User to Thing Group No Feature has not yet been implemented
Bulk Sensor Data Upload No Feature has not yet been implemented
User Maintenance - Deactivate a User No Feature has not yet been implemented
User Maintenance - Reset Credentials No Feature has not yet been implemented
