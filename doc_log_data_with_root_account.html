Introduction

Now that a Thing have been defined, it is time to create a little application to collect data and submit to the remote collector service.

Important: Normally only users created under the root account will submit data. This example is therefore mainly for demonstration purposes and help get users playing with some of the concepts early on. Using the root account is recommended to be done only from 1 or maybe 2 dedicated workstations identified for the purpose of administration of the root account. However, since all the required libraries may be installed, the workstation(s) are also Thing's in their own right, so there is no harm in creating a little application to record some basic workstation metrics.

There are some specific differences in using the root account vs using normal user accounts. The main differences are around permissions and token lifetime.

Feature Root Account User Account
Permissions Full access to all Thing's Specific access to a specific Thing or group of Thing's
Token Lifetime Absolute maximum token lifetime of 24 hours. Token lifetime can be set with no maximum (token can never expire)

Sample Application

A sample application is available at GitHub.

Once the repository is cloned, examine the example in the nix_load_average_logger directory.

Conclusion

Logging sensor data using the root account is not standard practice or even advised for production environments, but it does provide an easy way for hobbyists and those eager to explore the service's features.
